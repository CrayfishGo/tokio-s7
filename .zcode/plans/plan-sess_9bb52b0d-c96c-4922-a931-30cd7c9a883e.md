## 目标

把当前"流程式、状态分散"的连接管理重构为一个**纯函数状态机 (FSM)**，使每一次状态迁移都显式、可测试、对外可见，并补齐指数退避重连与状态变更通知。

---

## 一、FSM 设计（核心）

### 1. 状态 `ConnectionState`（重构现有枚举）

现有枚举有 3 个死状态（`Connected/Communicating` 从未赋值）。重构为 7 个真实状态，与握手子阶段一一对应：

```
Disconnected      初始/终结
Connecting        TCP 连接中
CotpHandshake     COTP CR 已发，等 CC
Negotiating       S7 Setup 已发，等响应
Ready             可收发命令（合并原 Negotiated/Communicating）
Reconnecting      退避等待中，准备重试（带 attempt 次数）
Disconnecting     已发 COTP DR，关 TCP 中
```

> 注：`Reconnecting { attempt: u32 }` 是带数据的变体，枚举因此从 `Copy` 降级为 `Clone`（watch 通道只需 `Clone`）。

### 2. 事件 `ConnEvent`（新增）

```rust
pub enum ConnEvent {
    Connect,                     // 用户调用 connect()
    TcpConnected,                // TCP connect 成功
    TcpFailed(S7Error),          // TCP 失败/超时
    CotpConfirmed,               // 收到 COTP CC
    CotpFailed(S7Error),         // COTP 被拒/关闭
    SetupOk(u16),                // Setup 协商成功，带 PDU 长度
    SetupFailed(S7Error),        // Setup 失败
    ConnectionLost(S7Error),     // 运行期连接断开
    Disconnect,                  // 用户调用 disconnect()
    DisconnectDone,              // DR 已发 + TCP 已关
    BackoffExpired,              // 退避计时结束
    Irrecoverable(S7Error),      // 不可恢复错误（如地址非法），停止重连
}
```

### 3. 迁移动作 `ConnAction`（新增，纯函数返回）

```rust
pub enum ConnAction {
    None,
    StartTcpConnect,
    SendCotpCr,
    SendSetupComm,
    EnterReady(u16),
    FailPending(S7Error),        // 把队列中待回复的命令标记失败
    StartBackoff(Duration),      // 计算退避时长
    SendDisconnect,
    CloseTcp,
    Stop,                        // actor 退出
}
```

### 4. 纯迁移函数 `transition(state, event) -> (ConnState, ConnAction)`

完整迁移表（所有"非法事件"返回 `ConnAction::None` 并保持原状态，可 log warn）：

| 当前 | 事件 | → 新状态 | 动作 |
|---|---|---|---|
| Disconnected | Connect | Connecting | StartTcpConnect |
| Connecting | TcpConnected | CotpHandshake | SendCotpCr |
| Connecting | TcpFailed | Reconnecting{1} / Disconnected | StartBackoff / FailPending+Stop |
| CotpHandshake | CotpConfirmed | Negotiating | SendSetupComm |
| CotpHandshake | CotpFailed | Reconnecting{n+1} | StartBackoff |
| Negotiating | SetupOk(pdu) | Ready | EnterReady(pdu) |
| Negotiating | SetupFailed | Reconnecting{n+1} | StartBackoff |
| Ready | ConnectionLost | Reconnecting{1} | FailPending+StartBackoff |
| Ready | Disconnect | Disconnecting | SendDisconnect |
| Reconnecting | BackoffExpired | Connecting | StartTcpConnect |
| Reconnecting | Irrecoverable | Disconnected | FailPending+Stop |
| Disconnecting | DisconnectDone | Disconnected | CloseTcp |

> 是否转 `Reconnecting` 由 `config.auto_reconnect` 决定；退避超过 `max_retries`（非0时）触发 `Irrecoverable`。纯函数把 `config` 和 `attempt` 作为参数传入，不读全局状态——**100% 可单测**。

## 二、退避策略（指数退避 + 最大重试）

### `S7Config` 新增字段

```rust
pub reconnect_interval_ms: u64,   // 初始退避，默认 1000
pub max_retries: u32,             // 0 = 无限重试，默认 0
pub backoff_factor: f32,          // 倍率，默认 2.0
pub backoff_cap_ms: u64,          // 退避上限，默认 30000
```

配 builder 方法 `with_reconnect_policy(interval, max, factor, cap)`。

### 计算函数（纯函数，在 connection.rs）

```
next_backoff(attempt) = min(initial * factor^attempt, cap)
```

当 `max_retries > 0 && attempt >= max_retries` → 返回 `None`（触发 Irrecoverable）。

## 三、状态变更通知（watch 通道）

- `S7Client` 内部持有 `watch::Sender<ConnectionState>`（actor 写）和对外暴露 `watch::Receiver<ConnectionState>`。
- actor 每次 `transition()` 后，若状态变化则 `tx.send(new_state)`。
- 公开 API：
  - `client.state() -> ConnectionState`（同步读最新值）
  - `client.state_receiver() -> watch::Receiver<ConnectionState>`（异步订阅）
  - `client.is_ready() -> bool`（便捷方法）
- `S7Client` 原来的 `connection_state` 字段直接换成 `watch::Receiver`，`connect()/disconnect()` 内的状态手动赋值删除，改由 watch 驱动。

## 四、文件改动清单

### 新增 `src/connection.rs`
- `ConnectionState`（从 client.rs 迁移 + 扩展）
- `ConnEvent`、`ConnAction`、`ConnTransition`（含 config/attempt 上下文）
- `transition()` 纯函数
- `next_backoff()` 纯函数
- `#[cfg(test)] mod tests`：覆盖迁移表所有路径 + 退避计算 + 非法事件忽略

### 改 `src/client.rs`
- 删除原 `ConnectionState` 定义（移至 connection.rs）
- `S7Client`：`connection_state` 字段 → `state_rx: watch::Receiver<ConnectionState>`，新增 `state_tx` 传给 actor
- `S7Config`：新增 4 个退避字段 + builder
- `connect()`：启动 actor 时把 `state_tx` 传入；握手结果仍用 oneshot，但状态由 FSM 驱动
- `disconnect()`：通过 FSM 确认当前在 `Ready`，发 Disconnect 命令
- `run_actor()`：用 FSM 重写主循环——每个阶段调 `transition()`，根据 `ConnAction` 执行副作用；重连用 `next_backoff()` + `sleep`（替换硬编码 2s）
- `handshake()` 内部拆成 `do_tcp_connect()` / `do_cotp_handshake()` / `do_setup_negotiate()` 三个函数，各自返回对应 `ConnEvent`（成功/失败），便于 FSM 精确定位失败点
- `serve_commands()`：进入前校验 FSM==Ready；每条命令处理完若出错，用 `ConnectionLost` 事件走 FSM

### 改 `src/lib.rs`
- 加 `pub mod connection;`

### 改 `examples/simple_client.rs`
- 演示 `client.state()` 查询（可选演示 watch 订阅）

## 五、关键约束

1. **纯函数优先**：`transition()` 和 `next_backoff()` 无 IO、无 async，所有外部输入（config/attempt）走参数，可独立单测。
2. **向后兼容**：`ConnectionState` 名字保留，仅新增变体；公共读 API（`state()`）保持可用。`connect()/disconnect()/read()/write()` 签名不变。
3. **非法操作被状态机拒绝**：如 `Disconnected` 状态下 `read()` 直接返回 `NotConnected`，而非卡在 `reply_rx.await`。
4. **不引入新依赖**：`watch` 在 tokio `sync` feature 内（已启用 `"full"`）。
